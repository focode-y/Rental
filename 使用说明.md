# Rental-template 使用说明

## 1. 项目说明
本项目是民宿代运营官网模板（静态站点），支持四种语言：
- 简体中文（zh）
- 繁体中文（hk）
- 日语（ja）
- 英语（en）

核心文件：
- `index.html`：页面结构与基础 SEO
- `styles.css`：样式
- `script.js`：多语言文案、交互逻辑、表单行为
- `data.js`：案例/评价/统计等展示数据

## 2. 常改项

### 2.1 联系方式与公司信息
在 `script.js` 的 `i18n` 中修改以下字段（四种语言都要改）：
- `footer_desc`
- `footer_hq`
- `footer_jp`
- `footer_hk`
- `footer_email`
- `footer_copy`

`index.html` 中也有少量静态占位文案（如页脚标题、电话占位文本），可按需改。

### 2.2 表单收件邮箱
`script.js`：
```js
const FORM_RECEIVER_EMAIL = "your-email@example.com";
```

页面中的表单 `action` 会由脚本自动同步为该邮箱。

### 2.3 案例与评价数据
编辑 `data.js`：
- `stats`
- `districts`
- `units`
- `reviews`

建议保留“示例”口径，避免放真实客户隐私信息。

## 3. 本地预览
可直接打开 `index.html`，或使用静态服务：

```bash
npx serve .
```

## 4. Cloudflare 安全发布（推荐）

### 4.1 一键安全发布
```powershell
npm run deploy:safe -- rental-template
```

### 4.2 发布前检查（单独执行）
```powershell
npm run check:security
```

### 4.3 GitHub 自动部署到 Cloudflare（推荐正式流程）
在 Cloudflare Pages 里连接 GitHub 仓库后，使用以下构建配置：
- Build command: `npm run build:publish`
- Build output directory: `publish`

这样每次 `git push` 到生产分支（如 `main`）都会自动部署到 Cloudflare。

## 5. 发布规则（强制）
- 禁止：`wrangler pages deploy .`
- 只允许：发布 `publish/` 目录
- 禁止上传：`.env*`、私钥、token、缓存目录

这些规则已写入：
- `AGENTS.md`
- `tools/prepublish-security-check.ps1`
- `tools/safe-deploy.ps1`

## 6. 常见问题

### Q1: 报错“Pages only supports files up to 25 MiB”
说明你上传了缓存目录（如 `.npm-cache`）。
解决：只用 `npm run deploy:safe -- <project-name>`。

### Q2: 页面出现 `???` 或乱码
通常是文件编码或历史替换导致。
解决：确保文件为 UTF-8，并优先修改 `script.js` 的 i18n 文案，不要用不稳定的批量正则替换中文文本。

### Q3: 如何避免误上传敏感内容
使用仓库内置脚本：
- `npm run check:security`
- `npm run deploy:safe -- <project-name>`
