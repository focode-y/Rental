# Rental 项目使用说明

本项目是纯静态页面（`index.html + styles.css + script.js + data.js`）。
修改内容时，优先改 `script.js`（固定文案）和 `data.js`（实绩数据与图片）。

> 重要：不要只改 `index.html` 文案。  
> 多语言最终由 `script.js` 里的 `i18n`（`zh/hk/ja/en`）控制，系统不会自动把中文改动翻译到日语/英语。  
> 需要同步修改对应语言键值，切换语言后才会正确显示。

## 0. 文案修改硬性检查清单（每次必做）

1. 先确认文案对应的 `data-i18n` 键名（如 `hero_title`、`hero_sub`）。
2. 在 `script.js` 的 `i18n` 中同时修改 `zh / hk / ja / en` 四个语言。
3. 若 `index.html` 有默认文案，也要同步修改，避免首屏短暂显示旧内容。
4. 修改后切换四种语言逐一检查，不允许只改中文。
5. 提交前自查：同一个键在四种语言都已更新、语义一致。

---

## 1. 文件结构与用途

- `index.html`
  - 页面结构（模块顺序、区块容器、表单、页脚）
- `styles.css`
  - 全站样式（配色、布局、响应式、按钮、卡片、导航）
- `script.js`
  - 三语文案字典（`zh/ja/en`）
  - 语言切换逻辑
  - 表单提交逻辑（邮箱在这里配置）
- `data.js`
  - “运营实绩”模块的动态数据
  - 包含：指标、商圈、案例卡片、客户评价

## 2. 先改这 2 处（最常用）

### 2.1 表单收件邮箱
- 文件：`script.js`
- 位置：顶部常量
- 字段：`FORM_RECEIVER_EMAIL`
- 示例：
```js
const FORM_RECEIVER_EMAIL = "admin@trytry.jp";
```

### 2.2 公司信息（页脚）
- 文件：`index.html`
- 位置：`<footer class="site-footer">` 内
- 建议修改内容：
  - 公司名
  - 公司简介
  - 总部地址
  - 电话
  - 邮箱

#### 公司信息具体修改位置（邮箱/电话/地址）

### A. 表单收件邮箱（访客提交后接收邮件）
- 主配置文件：`script.js`
- 字段：`FORM_RECEIVER_EMAIL`
- 示例：
```js
const FORM_RECEIVER_EMAIL = "new-email@example.com";
```
- 同步建议：把 `index.html` 中 `<form ... action="https://formsubmit.co/...">` 也改成同一个邮箱，避免维护时混淆。

### B. 联系方式卡片中的电话和邮箱（联系模块）
- 文件：`index.html`
- 区块：`<section id="contact">` 内 `.contact-quick`
- 需要同时改两处：
  - 链接值（`href="tel:..."` / `href="mailto:..."`）
  - 页面显示文本（`<strong>...</strong>`）

### C. 页脚中的地址、电话、邮箱
- 文件：`script.js`
- 位置：`const i18n = { ... }` 的各语言对象（`zh / hk / ja / en`）
- 常用字段：
  - 地址：`footer_hq`
  - 电话（日本）：`footer_jp`
  - 电话（香港）：`footer_hk`
  - 邮箱：`footer_email`
  - 公司简介：`footer_desc`
- 注意：项目是多语言，修改公司信息时请同步改对应语言字段，避免只在某个语言下生效。

### D. 公司名（页脚标题）
- 文件：`index.html`
- 区块：`<footer class="site-footer">`
- 位置：`<h3>Try Try</h3>`
- 说明：这里是固定文本，不走 i18n，改一次即可。

## 3. 每个模块内容替换位置

## 3.1 顶部导航 / Hero / 服务 / 优势 / 流程 / FAQ / 联系
- 文件：`script.js`
- 位置：`const i18n = { zh: {...}, ja: {...}, en: {...} }`
- 规则：
  - 中文改 `zh` 对应键
  - 日文改 `ja` 对应键
  - 英文改 `en` 对应键
- 常用键示例：
  - 导航：`nav_services`, `nav_advantage`, `nav_process`, `nav_faq`, `nav_cta`
  - Hero：`hero_title`, `hero_sub`, `hero_cta_1`, `hero_cta_2`
  - 服务：`services_title`, `s1_t/s1_d`, `s2_t/s2_d`, `s3_t/s3_d`
  - 优势：`adv_title`, `adv_sub`, `adv_l1~adv_l3`
  - 流程：`process_title`, `p1~p4`
  - 运营管理机构：`ops_title`, `ops_1~ops_8`
  - 语言支持：`lang_title`, `lang_desc`, `lang_1~lang_8`
  - 运营实绩标题：`bench_title`, `case_title`, `reviews_title`
  - FAQ：`faq_title`, `f1~f3`
  - 联系：`contact_title`, `contact_sub`, `form_name`, `form_contact`, `form_asset`, `form_btn`

## 3.2 “大阪在管30+房源｜运营实绩”模块（数据和图片）
- 文件：`data.js`
- 结构：
```js
window.PORTFOLIO_DATA = {
  zh: { stats: [], districts: [], units: [], reviews: [] },
  ja: { ... },
  en: { ... }
}
```

### A. 指标数据（顶部 4 卡）
- 字段：`stats`
- 每项格式：
```js
{ value: "30+", label: "大阪在管房源" }
```

### B. 商圈数据（中间 4 卡）
- 字段：`districts`
- 每项格式：
```js
{ title: "难波商圈", desc: "在管 12 套 | ..." }
```

### C. 真实案例（6 张房源图）
- 字段：`units`
- 每项格式：
```js
{
  title: "难波 1LDK A1",
  desc: "可住4人 | 近30天入住率 89%",
  image: "https://..."
}
```
- 只换图片时改 `image` 即可。

### D. 客户评价（文字卡片）
- 字段：`reviews`
- 每项格式：
```js
{ quote: "评价内容...", source: "业主A | 难波1LDK" }
```

## 6. 修改建议顺序（不容易出错）

1. 先改 `script.js` 文案（中/日/英）
2. 再改 `data.js` 的实绩数据和图片
3. 最后改 `styles.css` 样式
4. 每次改完强刷浏览器缓存：`Ctrl + F5`

---

## 7. 常见问题

### Q1: 改了没生效
- 先确认改的是正确文件
- 再 `Ctrl + F5` 强刷
- 检查有没有浏览器缓存或 CDN 缓存

### Q2: 中文正常，日文/英文没变
- 检查 `script.js` 的 `ja` / `en` 同名键是否同步修改

### Q3: 手机端样式错位
- 检查 `styles.css` 中 `@media (max-width: 768px)` 对应类是否被覆盖
